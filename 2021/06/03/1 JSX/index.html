JSX简介

在React中，我们会见到如下形式的值：

上述代码中的标签语法就是JSX，它既不是字符串也不是 HTML

JSX的一些常用规则如下：

  • 我们建议将JSX包裹在括号中，虽然这样做不是强制要求的，但是这可以避免遇到自动插入分号陷阱
  • 在 JSX 语法中，你可以在{}内放置任何有效的 JavaScript 表达式
  • 你可以通过使用引号，来将属性值指定为字符串字面量；也可以使用{}，来在属性值中插入一个 JavaScript 表达式
  • 如果没有子节点，你还可以使用自闭合的标签形式简写

JSX 标签里能够包含很多子元素

  • 因为 JSX 语法上更接近 JavaScript 而不是 HTML，所以 React DOM 使用 camelCase（小驼峰命名）来定义属性的名称，而不使用 HTML 属性名称的命名约定。例如，JSX 里的 class 变成了 className，而 tabindex 则变为 tabIndex

  • JSX 可防止注入攻击

所有的输入内容在渲染之前都被转换成了字符串。这样可以有效地防止 XSS（cross-site-scripting, 跨站脚本）攻击

JSX的实质

实际上，JSX 仅仅只是 React.createElement(component, props, ...children) 函数的语法糖。即 Babel 会把 JSX 转译成一个名为 React.createElement() 函数调用，该函数会返回一个React创建的普通{}对象，该对象的属性包含了你在JSX中设置的内容，React 通过读取这些对象，然后使用它们来构建 DOM 以及保持随时更新。这些对象被称为 “React 元素”（该对象就是我们所说的虚拟DOM）。

Babel除了将ES6转为ES5外，也可以用于转换JSX
