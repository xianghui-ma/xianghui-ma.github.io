错误边界

什么是错误边界？错误边界是一种 React 组件，这种组件可以捕获并打印发生在其子组件树任何位置的 JavaScript 错误，并且，它会渲染出备用 UI，而不是展现给用户一片爆红的错误

如何定义一个错误边界组件？

如果一个 class 组件中定义了 static getDerivedStateFromError()或componentDidCatch()这两个生命周期方法中的任意一个（或两个）时，那么它就变成一个错误边界

然后你就可以将错误边界组件作为一个常规组件去使用了：

注意：

  • 错误边界仅在后代组件的渲染期间、生命周期方法和整个组件树的构造函数中捕获错误。错误边界无法捕获后代组件中的下述错误：事件处理函数中的错误、异步代码（例如 setTimeout 或 requestAnimationFrame 回调函数）中的错误、错误边界自身抛出来的错误（并非它的后代组件）
  • 错误边界应该放在哪儿？你可以将它放在最顶层以处理整个APP的错误（一旦某个组件出错则整个APP便崩溃），也可以单独包裹在各个可能出错的组件外以处理单个组件的错误（一旦某个组件出错则仅影响该组件而不会致使整个APP出错）
  • 在开发环境下，为了展示错误信息以方便排错，错误边界只会展示一小段时间，然后照常爆红报错。仅在生产环境下，错误边界才会完全起作用

PureComponent（组件优化）

在这之前，我们都是使用Component，如：

但Component存在两个性能问题：

  • 只要调用setState，那么就会导致React重新渲染组件。这在大多数情况下是必要且正确的，但有时虽然我们调用了setState但数据没有任何变化，这时依旧会导致重新渲染（由于数据没变，故本来不用重新渲染），这便会引发性能问题
  • 父组件的重新渲染必然导致其中的子组件也被重新渲染。但有时子组件并没有用到父组件中的数据，或父组件传给子组件的props参数并没有改变。因此这时子组件本来不用重新渲染的。如下代码：
