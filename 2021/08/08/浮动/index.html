浮动

发表于 2021-08-08 更新于 2021-06-24 分类于 CSS
本文字数： 2.6k 阅读时长 ≈ 2 分钟

浮动设计的初衷是为了实现文字围绕图片的效果。后来才被广泛应用于布局

设置浮动

为元素设置float: left/right即可让元素浮动

浮动元素会被移出正常文档流，并被拉到容器左/右边。文本和行内元素会环绕浮动元素。如果让多个元素向同侧浮动，它们就会挨着排列。如下图所示：

容器折叠与清除浮动

浮动可能导致包含浮动元素的容器折叠。如下图所示：

要想让容器包含浮动元素，则可使用clear: both/left/right来清除浮动，clear表示是否将设置了该属性的元素移动到它前面（依照HTML源码书写顺序）的浮动元素之后（这就是“清除浮动”的真正含义）。clear只能对块级元素生效（浮动/非浮动都行）

  • clear: both移动到前面所有左和右浮动元素之后

  • clear: left移动到前面所有左浮动元素之后

  • clear: right移动到前面所有右浮动元素之后

明白了clear的含义，接下来我们就用他来解决容器折叠问题

1、方法一

在包含浮动元素的容器最后加上一个专门用于清除浮动的元素，并为该元素设置clear: both。这样，该元素将位于所有浮动元素后面，而该元素又是容器的子元素，所以容器定会包含该元素，从而解决了折叠问题

2、方法二

在包含浮动元素的容器上设置伪元素::after

3、方法三

为容器创建BFC（详细见下述“BFC”一节）

BFC(Block formatting context)

块级格式化上下文BFC是网页中的一块独立的区域，该区域与外部环境完全隔离互不影响，元素基于这块区域布局

给元素添加以下的任意属性值都会创建BFC：

  • float： left 或 right，不为 none 即可
  • overflow：hidden、auto 或 scroll，不为 visible 即可
  • display：inline-block、table-cell、table-caption、flex、inline-flex、grid 或 inline-grid。拥有这些属性的元素称为块级容器（block container）
  • position：absolute 或 position: fixed

这种隔离为创建 BFC 的元素做出了以下 3 件事情：

  • 包含了内部所有元素的上下外边距。它们不会跟 BFC 外面的元素产生外边距折叠
  • 包含了内部所有的浮动元素
  • 不会跟 BFC 外面的浮动元素重叠

其他特性

  • 在垂直方向上，两浮动元素之间的margin不会折叠
