定位

发表于 2021-08-08 更新于 2021-06-22 分类于 CSS
本文字数： 2.1k 阅读时长 ≈ 2 分钟

position属性的初始值是static（静态定位）。如果把它改成其他值，我们就说元素就被定位了。而如果元素使用了静态定位，那么就说它未被定位

固定定位

固定定位让元素相对视口定位（因此即使页面滚动，固定定位元素也不会动），此时视口被称作元素的包含块

给一个元素设置position: fixed就能将元素固定定位。还可以搭配四种属性一起使用：top、right、bottom 和 left。这些属性的值决定了固定定位的元素与包含块边缘的距离。设置这四个值还隐式地定义了元素的宽高。比如指定 left: 2em; right: 2em 表示元素的左边缘距离包含块左边 2em，右边缘距离包含块右边 2em。因此元素的宽度等于包含块总宽度减去4em。top、bottom 和包含块高度也是这样的关系

视口——浏览器窗口里网页可见部分的边框区域。它不包括浏览器的地址栏、工具栏、状态栏

因为固定定位元素从文档流中移除了，所以它不再影响页面其他元素的位置。别的元素会跟随正常文档流，就像固定定位元素不存在一样

绝对定位

绝对定位让元素相对最近的祖先定位元素定位，此时最近的祖先定位元素被称作元素的包含块

1、如果父元素未被定位，那么浏览器会沿着 DOM 树往上找它的祖父、曾祖父，直到找到一个定位元素，用它作为包含块； 2、如果祖先元素都没有定位，那么绝对定位的元素会基于初始包含块（initial containing block）来定位。初始包含块跟视口一样大，固定在网页的顶部

给一个元素设置position: absolute就能将元素绝对定位。同样可以搭配四种属性一起使用：top、right、bottom 和 left。这些属性的值决定了绝对定位的元素与包含块边缘的距离。设置这四个值还隐式地定义了元素的宽高。比如指定 left: 2em; right: 2em 表示元素的左边缘距离包含块左边 2em，右边缘距离包含块右边 2em。因此元素的宽度等于包含块总宽度减去4em。top、bottom 和包含块高度也是这样的关系

因为绝对定位元素从文档流中移除了，所以它不再影响页面其他元素的位置。别的元素会跟随正常文档流，就像绝对定位元素不存在一样

相对定位

相对定位让元素相对他原来的位置定位

给一个元素设置position: relative就能将元素相对定位。同样可以搭配四种属性一起使用：top、right、bottom 和 left。这些属性的值决定了相对定位的元素与他原来位置的偏移距离。设置这四个值不能隐式地定义元素的宽高（这与相对定位与绝对定位不同）。如position: relative、top: 1em、left: 2em表示将元素从原来的顶部边缘向下移动了1em，和将元素从它来的左侧边缘向右移动了2em

1、可以用 top 或者 bottom，但它们不能一起用（bottom 会被忽略）。同理，可以用 left 或 right，但它们也不能一起用（right 会被忽略） 2、常见的用法是使用 position: relative 给它里面的绝对定位元素创建一个包含块

相对定位元素从文档流中移除了，但他原来的位置仍被保留。所以其后面的盒子不会顶替它的位置

层叠上下文z-index

z-index只在定位元素上生效，不能用它控制静态元素（即position: static的元素，亦即非定位元素）

拥有较高 z-index 的元素出现在拥有较低 z-index 的元素前面。拥有负数 z-index 的元素出现在静态元素后面

浏览器会先绘制页面中所有非定位元素，然后绘制定位元素。当绘制定位元素时，是依照元素在 HTML 源码里出现的先后顺序进行绘制的。因此后绘制的定位元素会出现在先绘制的定位元素之前，此时我们便可使用z-index来调整层叠顺序

层叠上下文

给一个定位元素加上 z-index 可以创建层叠上下文。此时该定位元素就变成了层叠上下文的根。所有后代元素就是这个层叠上下文的一部分。（层叠上下文中可能还有层叠上下文，可以这样递归下去）
