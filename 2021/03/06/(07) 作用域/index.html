作用域

发表于 2021-03-06 分类于 ECMAScript
本文字数： 2.5k 阅读时长 ≈ 2 分钟

执行期上下文，作用域和作用域链是JS中的重要内容，本文将对他们进行具体阐述以说明它们是什么，它们间的联系以及作用域链所扮演的重要角色

执行期上下文

执行期上下文就是当前正在被执行的代码所处的运行环境，其分为

  • 全局执行期上下文（global execution context）
  • 函数执行期上下文（function execution context）

当执行全局（即函数体之外）中的代码时则位于全局执行期上下文中；当执行函数中的代码时则位于函数执行期上下文中。如下图所示：

对执行期上下文有了形象的认识后，我们接着讨论函数执行期上下文的形成过程：

  1. 遇见了一个函数调用语句
  2. 对函数进行预编译，预编译步骤如下：
  3. 1 首先创建该函数的executionContextObj空对象
  4. 2 初始化scopeChain
  5. 3 创建AO对象（见预编译一节）
  6. 4 确定this指向
  7. 函数预编译结束，开始执行函数中的代码

由上述我们便可推知全局执行期上下文的形成过程：

  1. 首先进行全局预编译，预编译步骤如下：
  2. 1 首先创建全局的executionContextObj空对象
  3. 2 初始化scopeChain
  4. 3 创建GO对象（见预编译一节）
  5. 4 确定this指向
  6. 全局预编译结束，从第一行开始执行全局中的所有代码

由上述函数和全局执行期上下文的创建过程并结合预编译一节。我们可知，AO/GO对象的创建仅是函数/全局执行期上下文创建过程中的一个步骤。即预编译阶段实际上就是创建执行期上下文，而在创建执行期上下文的过程中引擎构建了AO/GO对象

注意：函数每调用一次就会创建一个新的执行期上下文，函数执行完毕后执行期上下文也随之被销毁，且每次创建的执行期上下文都与上一次所创建的执行期上下文无关

作用域与作用域链

作用域就是AO/GO对象，而作用域链就是executionContextObj中的scopeChain。而JS解释器使用的是词法作用域，所谓词法作用域就是——要求将内层函数的作用域绑定到他的第一个父级函数的作用域上。倘若第一个父级函数又是其他函数的内层函数，那么便以此类推继续将它的作用域绑定到他的第一个父级函数的作用域上……，这样便形成了一种链的形式——即作用域链。所以scopeChain满足下列等式：

如果将作用域链想象为一个栈的话，那么当前函数的AO始终位于栈顶；GO始终位于栈底；而当前函数的所有父级函数的AO位于它们俩之间。如：

其中three()函数的scopeChain = three's AO + two's AO + one's AO + GO；
two()函数的scopeChain = two's AO + one's AO + GO；
one()函数的scopeChain = one's AO + GO；
全局的scopeChain = GO；

需要特别注意的是：scopeChain中的AO/GO实际上是一个引用，该引用指向正真的AO/GO。即如two()函数中的one's AO与three()函数中的one's AO都只是one()函数的AO的引用，所以实际上这两个函数的one's AO是同一个

LHS与RHS查询

至此，我们已经明晰了作用域链。那么作用域链有什么用呢？这就是我们接下来需要讨论的问题。实际上，作用域链的作用就是用来进行标识符查询以对标识符进行相应操作（如变量赋值，函数调用等）

而标识符查询又分为：

  • LHS查询。LHS查询就是找到标识符容器本身以将值放入容器
  • RHS查询。RHS查询就是查找某个标识符的值

上述代码中的LHS查询有：

  1. 查询变量c以将函数调用的返回值赋予c
  2. 查询形参a以将实参2赋予形参
  3. 查询变量b以将a的值赋予b

而RHS查询有：

  1. 查询foo以得到foo中存储的函数引用，从而调用foo函数
  2. 查询var b = a中的变量a以得到它的值，从而将值赋予b
  3. 查询return a + b;中的变量a与b，从而相加返回
