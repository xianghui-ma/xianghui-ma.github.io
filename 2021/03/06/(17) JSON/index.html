(12)JSON

JSON（JavaScript Object Notation-JavaScript对象表示法）是一种表示值和对象的数据格式（或者说JSON就是一个用于表示值/对象的字符串）。其最初为JS所有，但后来许多其他语言也都内置有JSON。因此，借助JSON，我么可以十分方便在客户端与服务器端间交换数据。

例如，我们想将一个对象形式的数据通过网络进行传输，但是没法直接将一个对象在网络中进行传输呀！但是我们可将对象转换为JSON字符串，这样便能进行传输了

JSON基本使用

JS为JSON提供了两个主要方法：

  • JSON.stringify(value)其用于将value转换为JSON,返回一个JSON字符串。其只序列化对象自身的可枚举的属性
  • JSON.parse(JSON)其用于将JSON转换为原来的形式（如原来是一个对象则JSON会被转换为对象；原来是一个Boolean值则JSON便被转换为Boolean值……）

其中value可为下列数据类型：

  • 对象
  • 数组
  • 字符串
  • 数字
  • Boolean值
  • null

undefined不被JSON所支持

通过stringify获得的结果也被称为JSON序列化对象

但是，在对象（包括嵌套的内层对象）序列化过程中，stringify有如下特殊特性：

  • 属性值为一个函数的属性不会被stringify序列化
  • 属性值为undefined的属性不会被stringify序列化；数组中的某个值为undefined时该值会被stringify序列化为null
  • 对象中的属性值为循环引用时，序列化会报错Uncaught TypeError: Converting circular structure to JSON

JSON方法的更多特性

上述只是stringify与parse的基本常用用法，下面我们对其进行更为完整的探究：

stringify的完整语法是：JSON.stringify(value, replacer, space])

replacer参数用于告诉stringify你想序列化哪些属性。其值为数组或函数：

  • replacer为数组时

此时，replacer只会序列化数组中指定的属性，且该规则还会作用于内层嵌套对象

stringify只序列化了数组中指定的title与participants属性。同时，由于数组中没有包含name属性，因此内层嵌套对象中的name属性不会被序列化

  • replacer为function(key, value)函数时

stringify会为对象（包括内层嵌套对象）中的每一个键值对调用该函数，且该函数必须有返回值以用于告诉stringify当前遍历出的这个属性是否需要被序列化。若返回undefined则表明当前属性不需要被序列化；若返回value则表明当前属性需要被序列化。需要注意的是：只能返回undefined或value
