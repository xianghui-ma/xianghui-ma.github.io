脚本化CSS

可通过下述两种方式实现CSS脚本化：

  • Element.style
  1. 该方式可读/写。写入值时，值应用字符串表示
  2. 该方式只能读/写HTML标签中的style属性。通过此种方式访问那些没在style属性中设置的CSS属性的值时会返回""
  3. 该方式的返回值是一个对象
  4. 访问float属性时应使用cssFloat（因为float是JS的保留字）即Element.style.cssFloat
  5. 组合属性应该用小驼峰读/写，即应该是Element.style.backgroundColor而非Element.style.background-color
  6. 复合属性必须拆解为单个进行读/写，即如要读写border时应为Element.style.borderStyle,Element.style.borderWidth,Element.style.borderColor

由上可知，Element.style操作的仅是标签的style属性，由于background-color不是设置在style属性上的，故不能通过此种方式进行读写

  • window.getComputedStyle(Element, [pseudo])
  1. 其返回一个对象，且其只读
  2. [pseudo]为Element中的伪元素，该参数可选，不需要获取伪元素的样式时可省略该参数或传入null
  3. 与Element.style不同的是其返回的是经浏览器计算后的最终用于Element上的所有样式
  4. 其返回值均为绝对值，如设置height:10em则window.getComputedStyle(Element).heigth返回160px
  5. 复合/组合属性的书写规范同style

但是在IE中没有上述方法，故在IE中应使用Element.currentStyle。除了CSS样式的值不是绝对值（你设置什么值便返回什么值）外，其其他特性同getComputedStyle
