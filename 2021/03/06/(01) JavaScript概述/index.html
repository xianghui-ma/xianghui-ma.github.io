(01)JavaScript概述

发表于 2021-03-06 分类于 ECMAScript
本文字数： 1.4k 阅读时长 ≈ 1 分钟

JavaScript的由来

JavaScript诞生于1995年，当时主要目的是表单验证。在JS诞生前，必须将表单数据发送至服务器后才能验证用户是否输入了无效值或是否未填写必填项等，但是，当时使用的是电话拨号上网，因此与服务器端的每一次数据交互都会耗费大量时间，因此上述表单验证方式无疑是耐心杀手。为在客户端解决表单验证问题，JavaScript诞生。当时，Netscape职员布兰登•艾奇(Brendan Eich)着手为即将发布的Netscape Navigator 2 开发一种名为 LiveScript 的脚本语言(JavaScript最开始叫LiveScript)。为在发布前完成，Netscape便与Sun公司合作开发LiveScript，开发完成后为搭乘sun的Java顺风车于是将LiveScript更名为JavaScript

ECMAScript的由来

在Netscape的JavaScript取得极大成功的同时，微软IE为与Navigator竞争，便在IE3中加入了名为JScript的JavaScript实现(命名为JScript 是为了避开与 Netscape 有关的授权问题)。那么这就意味着出现了两个不同的JavaScript版本，为进行统一与规范，因此ECMA于1997年发布了ECMAScript标准(即ECMA-262)。该标准主要是对JavaScript的语法与语义进行规范

JavaScript的组成

JavaScript由如下三部分组成：

  • ECMAScript
  • DOM
  • BOM

其中ECMAScript是JavaScript的核心，其定义了JavaScript的语言基础；DOM与BOM是JavaScript的扩展。下面分别对上述三部分进行阐释：ECMAScript与浏览器不存在依赖关系，其是独立的，因为ECMAScript是一个标准，JavaScript是ECMAScript的实现，而虽然浏览器不同，但各个浏览器的JavaScript都必须要按照这个标准实现啊！不存在差异；具体地，ECMAScript对如下内容进行了规范:

  • 语法
  • 类型
  • 语句
  • 关键字
  • 保留字
  • 操作符
  • 对象

DOM提供访问和操作网页内容的方法和接口。其将整个HTML页面映射为一个多层节点结构，HTML中的元素便为该结构中的节点。然后借助DOM提供的API，我们便可对节点进行增删改查等操作，从而实现对页面结构及内容的控制。

BOM提供与浏览器交互的方法和接口。但由于缺乏规范，因此不同浏览器中的BOM可能有所差异。下面就是BOM中的一些部分：

  • 弹出新浏览器窗口的功能；
  • 移动、缩放和关闭浏览器窗口的功能；
  • 提供浏览器详细信息的 navigator 对象；
  • 提供浏览器所加载页面的详细信息的 location 对象；
  • 提供用户显示器分辨率详细信息的 screen 对象；
  • 对 cookies 的支持；
  • 像 XMLHttpRequest 和 IE 的 ActiveXObject 这样的自定义对象

JavaScript特点

  • 解释性语言

无论何种语言，只有被编译为0/1代码后才能被计算机执行。对于编译型语言而言，其将代码编译为一个目标文件后执行目标文件；对于解释型语言而言，其将一行代码编译为0/1后马上执行，接着继续下一行，不生成目标文件。

  • JS引擎是单线程的

计算机中的异步是指同一时间干多件事，同步是指同一时间仅能干一件事。

  • 遵循ECMAscript标准

即由ECMA制定标准，然后各浏览器厂商按照该标准实现JS，从而实现统一。
