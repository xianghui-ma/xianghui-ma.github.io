查找元素

选取文档元素

选取文档元素的方法有：

  1. document.getElementById("id属性值")
  2. document.getElementsByClassName("class属性值")
  3. document.getElementsByTagName("标签名")
  4. document.getElementsByName("name属性值")
  5. document.querySelector("css选择器")
  6. document.querySelectorAll("css选择器")

上述方法均来源于Document.prototype，而document实例继承了这些方法

其中1-4说明如下：

  • 方法名中有Elements的返回的均是符合条件的元素组成的类数组
  • 当文档中有多个同值id（这是不规范的）时getElementById只会选择第一个
  • 1-4的方法的结果都是实时的——即结果会随着程序的增删改操作做相应的增删改
  • getElementsByTagName与getElementsByClassName不仅定义在了Document.prototype上，而且它们也定义在了Element.prototype,这就意味着元素节点也能使用这两个方法：

其中5和6的说明如下：

  • 它们的结果不是实时的
  • querySelector仅选择符合条件的第一个元素，而querySelectorAll会选择符合条件的所有元素（类数组）

遍历node tree中的节点

节点的遍历方式有：

  1. node.parentNode获取某个节点的父节点（直接父节点）
  2. node.childNodes获取某个节点的子节点（直接子节点）
  3. node.firstChild获取某个节点的第一个子节点（直接子节点）
  4. node.lastChild获取某个节点的最后一个子节点（直接子节点）
  5. node.nextSibling获取某个节点的前一个兄弟节点
  6. node.previousSibling获取某个节点的后一个兄弟节点

上述方式的说明：

  • 上述方式会遍历所有类型的节点
  • 上述都是继承自Node.prototype
  • 上述结果都是实时的

遍历node tree中的元素节点

元素节点的遍历方式有：

  1. node.parentElement获取某个节点的父元素（直接父元素）节点
  2. node.children获取某个节点的所有子元素（直接子元素）节点，类数组返回
  3. node.childElementCount获取某个节点的子元素（直接子元素）个数，等价于node.children.length
  4. node.firstElementChild获取某个节点的第一个子元素（直接子元素）
  5. node.lastElementChild获取某个节点的最后一个子元素（直接子元素）
  6. node.previousElementSibling获取某个节点的上一个兄弟元素
  7. node.nextElementSibling获取某个节点的下一个兄弟元素

上述方式的说明：

  • 上述方式仅遍历HTML元素节点，元素节点之外的节点不会被遍历
  • 上述parentElement继承自Node.prototype；其他均继承自Element.prototype
  • 上述结果都是实时的

节点的四个属性

  • nodeName

用于获取节点名称，各类型的节点名称如下：

  1. document.nodeName为#document
  2. 元素节点.nodeName为该元素标签名（大写形式）
  3. 文本节点.nodeName为#text
  4. 注释节点.nodeName为#comment
  5. 属性节点.nodeName为该属性的名字（如id/class...）
  6. 文档片段节点.nodeName为#document-fragment
  • nodeValue

用于获取节点的值，各类型的节点值如下：

  1. document.nodeValue为null
  2. 元素节点.nodeValue为null
  3. 文本节点.nodeValue为文本内容
  4. 注释节点.nodeValue为注释内容
  5. 属性节点.nodeValue为该属性的值
  6. 文档片段节点.nodeValue为null
  • nodeType

用于获取节点类型，各类型节点的nodeType值如下：

  1. document.nodeType为9
  2. 元素节点.nodeType为1
  3. 文本节点.nodeType为3
  4. 注释节点.nodeType为8
  5. 属性节点.nodeType为2
  6. 文档片段节点.nodeType为11
  • attributes

用于获取元素节点的所有属性（以属性节点组成的类数组返回）

注意：
上述前三个属性位于Node.prototype中，因此所有节点都有这些属性；但是attributes位于Element.prototype中，因此只有元素节点拥有该属性

除了上述所有知识点外，再补充一些小点：

  1. document.getElementsByTagName("*")将获取到所有元素节点
  2. document.body/head可直接获取body或head元素节点
