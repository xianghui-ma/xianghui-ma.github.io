事件处理模型

DOM Events标准规定了两种事件处理模型——事件冒泡与事件捕获

事件冒泡

事件冒泡的规则为：当某个元素的A事件被触发后，绑定在该元素上的对于A的事件处理函数会首先被执行，然后该元素的父元素的A事件处理函数（如果有的话）被执行……，这样依次向上

当点击p元素后，事件处理函数的执行顺序为：

  1. p元素的click处理函数执行
  2. div的click处理函数冒泡执行
  3. form的click处理函数冒泡执行
  4. ……如此向上直到window

综上可见：事件冒泡是在HTML结构上由子元素向父元素并依次顺着其所有祖先元素进行的

并非所有事件都能冒泡，如focus事件便不能冒泡

事件捕获

与事件冒泡正好相反：事件捕获是先从事件触发元素的祖先元素开始，然后逐层向内捕获至事件触发元素

当点击div#content元素后，事件处理函数的执行顺序为：

  1. 从window开始向下捕获执行…
  2. div#outter的click处理函数捕获执行
  3. div#inner的click处理函数捕获执行
  4. div#content的click处理函数执行

事件传播3阶段

上述只是分开讨论了事件冒泡与捕获。在对他们有一定认识之后，现在我们将结合事件捕获和冒泡以讨论浏览器中的事件传播阶段，从而明晰事件冒泡与捕获谁先谁后

DOM Events标准规定了浏览器的事件传播阶段。其具体分为三个阶段并按照下述顺序进行：

  1. 事件捕获阶段
  2. 事件触发元素本身的事件处理函数执行阶段
  3. 事件冒泡阶段

注意：第2阶段既不属于事件捕获阶段，也不属于事件冒泡阶段。第2阶段就是普通的事件触发所导致的回调函数执行

下图对此做了更为详细的描述

因此，当你点击<td>时：首先进行的是事件捕获；然后进行的是事件触发元素本身的事件处理函数执行；最后进行的是事件冒泡

需要注意的是，下述几种事件绑定方式没有事件捕获阶段，它们仅有第2与第3阶段：

  • node.onclick=回调函数
  • 直接在HTML元素上设置事件属性，如<div onclick="alert('div')">
  • node.addEventListener("click", 回调函数, boolean值)中的第三个参数为false或省略第三个参数时
  • node.attachEvent("onclick", 回调函数)

同时还需要注意的是：

  • 第2阶段时，回调函数的执行顺序与它们在程序中的绑定顺序一致
  • 你绑定事件时，只能选择一个事件处理模型——即你选了事件冒泡就没有事件捕获，你选了事件捕获就没有事件冒泡

按照事件传播三阶段，当点击div#content元素时，事件按下属顺序执行：

  1. 首先进入捕获阶段，打印顺序为outter-capturing => inner-capturing
  2. 捕获到事件触发元素本身时，便进入第2阶段，此时回调函数按绑定顺序执行content-bubbling => content-capturing
  3. 事件触发元素本身的回调执行完后便进入冒泡阶段，打印顺序为inner-bubbling => outter-bubbling

若在for循环中先绑定捕获回调，再绑定冒泡回调，则第2阶段的输出顺序为content-capturing => content-bubbling,其他阶段输出顺序不变。因为第2阶段是按照绑定先后顺序执行

阻止冒泡和默认事件

如果我们不想事件冒泡，则可通过事件对象.stopPropagation()来阻止事件冒泡。但是在低版本IE中要使用事件对象.cancelBubble=true来取消事件冒泡

除此之外，浏览器中还有许多默认事件，如：点击一个链接会导致跳转；点击表单提交按钮会提交表单等等。如果你不想浏览器采取这些默认行为，那么可通过取消默认事件达到目的。取消默认事件的方式如下：

  • 在回调函数中返回false，即return false。但该种方式仅在以node.onclick=回调函数形式绑定的回调函数中有效
  • 事件对象.preventDefault()。但是低版本IE中需要通过事件对象.returnValue=false来实现

事件类型

鼠标事件

  • click鼠标点击事件。click=mousedown+mouseup，DOM3规定只有鼠标左键才能触发click
  • contextmenu右键出菜单事件。当鼠标在某个元素上右击时触发
  • mouseover/mouseout鼠标移入/移出事件。当鼠标移入/移出某个元素时触发。在H5中它们分别更名为mouseenter/mouseleave
  • mousedown/mouseup鼠标按下/弹起事件。当鼠标在某个元素上按下/弹起时触发。鼠标左/右键都能触发该事件
  • mousemove鼠标移动事件。当鼠标移动时触发

在元素的content--border区域内才会触发上述事件，margin内不会触发上述事件

某些事件，鼠标左/右键都能触发，那怎么判断是左键还是右键触发的呢？可通过事件对象.button来判断。其会返回一个数字：左键对应0，中键对应1，右键对应2

键盘事件

  • keydown/keyup当键盘按键按下/松开时触发，其响应所有按键。且若按住某个按键不放则keydown会一直触发
  • keypress其不会响应所有按键。除此之外与keydown相同。下表展示了在不同的浏览器中哪些按键能触发keypress
按键 IE Firefox Opera Chrome&Safari
数字&字母键,+,-,*,/,Enter,空格 yes yes yes yes
Alt,Ctrl,Shift,系统键 no no yes no
Esc yes yes yes no
Fn,方向键,Insert,Delete,Home,End,翻页键,Backspace no yes yes no

键盘事件的事件对象中有几个属性值得注意：

  • which

在keydown与keypress下均可工作。在keyPress下其值为按键对应字符的Unicode编码；在keydown下其值为按键的Unicode key code编码

  • keyCode

在keydown与keypress下均可工作。在keyPress下其值为按键对应字符的Unicode编码；在keydown下其值为按键的Unicode key code编码。在chrome/safari/IE9及以上中，keyCode与charCode值一样

  • charCode

仅在keyPress下工作。在chrome/safari/IE9及以上中，keyCode与charCode值一样

综上所述，如果你想读取按键对应字符的Unicode编码，那么应选择keyPress事件及which属性（其兼容性很好）

表单（或称文本）事件

  • change对于input type="text"而言，当文本输入区失去焦点且本次输入与上一次内容不同时才会触发该事件。对于input type=""checkbox/radio而言，当做出与上一次不同的选择时触发
  • input只要文本框中的内容发生变化则立即出发。这跟百度搜索输入框是一致的——任何一个字符的输入都会触发该事件，而每一次触发都会获取文本框中的值以进行联想
  • focus/blur在input type="text"获得/失去焦点时触发

其他事件

  • scroll当滚动条发生滚动时触发该事件。其存在于window和具有滚动条的元素上
