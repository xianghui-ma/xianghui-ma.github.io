8 状态提升

什么是状态提升？通常，多个组件需要共享相同的变化数据，这时我们建议将这些数据（或者说是state）提升到距这些组件最近的共同父组件中的state中去

如何将这些子组件中的共享数据传给父组件的state呢？方法是，我们在父组件中定义一个修改父组件的state的方法（该方法中的this已绑定为父组件的实例），然后将该方法通过子组件的props传给子组件，然后在子组件中调用传入的方法，从而将子组件中的变化数据传递给父组件的state

父组件又如何将共享数据传递给那些需要该数据的子组件呢？通过子组件的props传递给子组件（即自上而下的数据流）

state与生命周期初识

什么是state与生命周期

state就是组件中的数据，而这些数据是变化的，只要state中的内容发生变化（即组件的状态发生变化），那么React就会自动调用组件中的render方法来更新页面数据

state仅在类组件中使用。函数组件需要借助于钩子

在“第2章”中我们说到“更新 UI 唯一的方式是创建一个全新的元素，并将其传入 ReactDOM.render()”。现在我们可通过state来更新组件中的数据（即更新页面内容）。这儿对“第2章”中的页面计时器进行改写

上述代码的执行逻辑为：

  1. 当 <Clock /> 被传给 ReactDOM.render()的时候，React 会调用 Clock 组件的构造函数。因为 Clock 需要显示当前的时间，所以它会用一个包含当前时间的对象来初始化 this.state。我们会在之后更新 state
  2. 之后 React 会调用组件的 render() 方法。这就是 React 确定该在页面上展示什么的方式。然后 React 更新 DOM 来匹配 Clock 渲染的输出
  3. 当 Clock 的输出被插入到 DOM 中后，React 就会调用 ComponentDidMount() 生命周期方法。在这个方法中，Clock 组件向浏览器请求设置一个计时器来每秒调用一次组件的 tick() 方法
  4. 浏览器每秒都会调用一次 tick() 方法。 在这方法之中，Clock 组件会通过调用 setState() 来计划进行一次 UI 更新。得益于 setState() 的调用，React 能够知道 state 已经改变了，然后会重新调用 render() 方法来确定页面上该显示什么。这一次，render() 方法中的 this.state.date 就不一样了，如此以来就会渲染输出更新过的时间。React 也会相应的更新 DOM
  5. 一旦 Clock 组件从 DOM 中被移除，React 就会调用 componentWillUnmount() 生命周期方法，这样计时器就停止了

JSX简介

在React中，我们会见到如下形式的值：

上述代码中的标签语法就是JSX，它既不是字符串也不是 HTML

JSX的一些常用规则如下：

  • 我们建议将JSX包裹在括号中，虽然这样做不是强制要求的，但是这可以避免遇到自动插入分号陷阱
  • 在 JSX 语法中，你可以在{}内放置任何有效的 JavaScript 表达式
  • 你可以通过使用引号，来将属性值指定为字符串字面量；也可以使用{}，来在属性值中插入一个 JavaScript 表达式
  • 如果没有子节点，你还可以使用自闭合的标签形式简写

JSX 标签里能够包含很多子元素

  • 因为 JSX 语法上更接近 JavaScript 而不是 HTML，所以 React DOM 使用 camelCase（小驼峰命名）来定义属性的名称，而不使用 HTML 属性名称的命名约定。例如，JSX 里的 class 变成了 className，而 tabindex 则变为 tabIndex

  • JSX 可防止注入攻击

所有的输入内容在渲染之前都被转换成了字符串。这样可以有效地防止 XSS（cross-site-scripting, 跨站脚本）攻击

JSX的实质

实际上，JSX 仅仅只是 React.createElement(component, props, ...children) 函数的语法糖。即 Babel 会把 JSX 转译成一个名为 React.createElement() 函数调用，该函数会返回一个React创建的普通{}对象，该对象的属性包含了你在JSX中设置的内容，React 通过读取这些对象，然后使用它们来构建 DOM 以及保持随时更新。这些对象被称为 “React 元素”（该对象就是我们所说的虚拟DOM）。
