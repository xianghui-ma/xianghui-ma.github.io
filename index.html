  • pageXOffset与pageYOffset

使用方式为：window.pageXOffset/pageYOffset。用于获取文档相对于窗口坐标系水平/垂直方向的滚动距离（或者说获取滚动条在水平/垂直方向的滚动距离）。其返回的是一个数字，单位是像素

但低版本IE不兼容上述方法，所以在IE中应使用：document.body.scrollLeft/scrollTop与document.documentElement.scrollLeft/scrollTop

  • innerWidth和innerHeight

使用方式为：window.innerWidth/innerHeight。用于获取视口的宽/高。需要注意的是其会将滚动条所占据的宽/高度算进去

但低版本IE不兼容上述方法，所以在IE中应使用：document.body.clientWidth/clientHeight与document.documentElement.clientWidth/clientHeight。但是clientWidth/clientHeight不包含滚动条所占据的宽/高度

(6)定时器与循环

发表于 2021-03-06 分类于 DOM与BOM
本文字数： 2.2k 阅读时长 ≈ 2 分钟

本节讨论window下的setTimeout与setInterval方法的使用及原理。并借此阐释事件循环机制

基本使用

  • setTimeout与clearTimeout

setTimeout的传参形式为setTimeout(待执行函数, 延时时间<一般以毫秒为单位>)。延时时间到后，待执行函数会被调用执行，且仅执行一次。该函数还会返回一个ID用于标记你设置的这个定时器，你可将这个ID传给clearTimeout以取消该定时器

  • setInterval与clearInterval

setInterval的传参形式为setInterval(待执行函数, 间隔时间<一般以毫秒为单位>)。每过间隔时间后待执行函数便被执行一次。该函数还会返回一个ID用于标记你设置的这个定时器，你可将这个ID传给clearInterval以取消该定时器。如果不取消则会一直执行

实际上，上述API的时间是不准确的——即上述两个函数并不能保证在延时时间/间隔时间到后立即执行回调函数；所以上述API的真正含义是：在延时时间/间隔时间到后立即将回调函数放入任务队列中并等待被调用栈执行。在下面的事件循环中，我们将对其进行深入探讨

脚本化CSS

可通过下述两种方式实现CSS脚本化：

  • Element.style
  1. 该方式可读/写。写入值时，值应用字符串表示
  2. 该方式只能读/写HTML标签中的style属性。通过此种方式访问那些没在style属性中设置的CSS属性的值时会返回""
  3. 该方式的返回值是一个对象
  4. 访问float属性时应使用cssFloat（因为float是JS的保留字）即Element.style.cssFloat
  5. 组合属性应该用小驼峰读/写，即应该是Element.style.backgroundColor而非Element.style.background-color
  6. 复合属性必须拆解为单个进行读/写，即如要读写border时应为Element.style.borderStyle,Element.style.borderWidth,Element.style.borderColor

(16)严格模式

发表于 2021-03-06 分类于 ECMAScript
本文字数： 2.4k 阅读时长 ≈ 2 分钟

本节将从严格模式的由来；怎样触发严格模式；严格模式与非严格模式的区别展开讨论

严格模式由来

在ECMAScript 5之前，ECMA更新JavaScript的方式是：加入新特性的同时不会对旧的特性做任何改变。如此以来，JavaScript便不会存在兼容性问题（因为旧特性的保留使得那些使用旧特性书写的代码也能照常运行）。但是，从ECMAScript 5开始，ECMA在加入性特性的同时还对旧特性做了更正，因此这就致使用旧特性书写的代码不能照常运行，从而引发了JavaScript的兼容性问题。

为保证使用旧特性书写的代码依然能照常运行，因此便诞生了严格模式这一开关：打开它时，浏览器就会使用严格模式执行代码，此时使用旧特性书写的代码便不能正常运行了；关闭它时，浏览器便按照非严格模式执行代码，此时使用旧特性书写的代码依然能正常运行。从而，通过严格模式这一开关便能解决上述兼容性问题了！
